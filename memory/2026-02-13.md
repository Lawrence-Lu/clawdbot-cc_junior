## 任务5：日报生成流程优化完成

### 问题复盘
今天日报生成过程中出现了重复步骤，原因是 Token 过期后没有第一时间使用已验证的方法，而是尝试了多种新方式导致效率低下。

### 优化方案
与用户确认了完整的日报生成流程和分叉处理逻辑：

**主流程（18:40 触发）**:
1. 检查是否工作日 (holiday_checker.js) → 非工作日直接停止
2. 发送消息："请提供飞书 User Token"
3. 获取今日日程 (calendar API)
4. 从日程描述提取文档名称，匹配会议纪要
5. 生成日报草稿
6. **发送草稿给用户确认**（30分钟超时提醒一次）
7. 用户确认后写入飞书日报文件夹

**分叉处理**:
| 场景 | 处理方式 |
|:---|:---|
| 30分钟未回复 Token | 提醒："还需要 Token 吗？" |
| 今日无日程 | 询问用户工作内容 |
| 会议纪要读取失败 | 标注"请补充" |
| 30分钟未回复确认 | 提醒一次 → 再30分钟无回复则取消，第二天可回溯 |
| 写入飞书失败 | 飞书提醒用户 |

### 新增文件
- `daily_report_v2.js`: 生成草稿，等待确认
- `daily_report_confirm.js`: 确认后写入飞书
